# Triage Labels

This repo uses the default triage label vocabulary.

| Canonical role | Label |
| --- | --- |
| Needs maintainer evaluation | `needs-triage` |
| Waiting on reporter/user information | `needs-info` |
| Fully specified and ready for an agent | `ready-for-agent` |
| Ready for human implementation | `ready-for-human` |
| Will not be actioned | `wontfix` |

Skills that create or triage issues should use these labels exactly.

If the GitHub repository does not have these labels yet, create them before running issue automation at scale.
